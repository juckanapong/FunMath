# แน่ใจนะ ว่าคิดเป็น (Math Duel — ศึกสองทีม)

สื่อเกมการเรียนรู้คณิตศาสตร์แนว Camera-based Interaction / Embodied Learning
สำหรับนักเรียนระดับประถมศึกษา · พัฒนาโดย **ครูกุ้ง**
แรงบันดาลใจจากเกม "เก่งคณิต คิดสนุก" ของเพจคุณครูใจดี

---

## 1. แนวคิดของเกม (Game Concept)

นักเรียนแบ่งเป็น 2 ทีม (ทีมแดง–ทีมน้ำเงิน) ส่งตัวแทนทีมละ 1 คน
มายืนหน้ากล้องพร้อมกัน ฝั่งซ้ายและฝั่งขวาของจอ แข่งกันตอบโจทย์คณิตศาสตร์
โดยใช้ **การยกมือจริง ๆ** ไปแตะลูกบอลคำตอบบนจอ ภายในเวลาการแข่งที่ตั้งไว้
(60/90/120 วินาทีต่อแมตช์) ทีมที่ตอบถูกก่อนได้คะแนน และได้สิทธิ์แตะ
**กล่องปริศนา** รัว ๆ เพื่อเก็บคะแนนพิเศษเข้าทีมตัวเอง

จุดเด่นเชิงการเรียนรู้:

- **Embodied Learning** — ใช้ร่างกายเคลื่อนไหวควบคุมเกม ช่วยการจดจำและการมีส่วนร่วม
- **Gamification** — คะแนน เวลา ดาวโบนัส ภารกิจท่าทาง และการแข่งขันเป็นทีม
- **Immediate Feedback** — เสียงและภาพตอบสนองทันทีเมื่อตอบถูก/ผิด พร้อมเฉลย
- **Differentiation** — ปรับชนิดโจทย์ (บวก ลบ คูณ หาร คละ) ระดับความยาก 3 ระดับ และระยะเวลาแข่งได้

## 2. ลำดับหน้าจอ (Game Flow)

1. **หน้าปก** — ภาพปกเกม (`cover.jpg`) พร้อมปุ่ม "เข้าสู่เกม" (เพลงประกอบเริ่มเล่นที่นี่)
2. **หน้าตั้งค่า** — เลือกโจทย์ (บวก/ลบ/คูณ/หาร/คละ) ระดับความยาก ระยะเวลาแข่ง (60/90/120 วินาที)
   กดปุ่ม **"วิธีเล่น"** เพื่อดูกติกาและกล่องปริศนาได้ทุกเมื่อ (มีปุ่ม "เปิดกล้อง" ให้ทดสอบกล้องล่วงหน้า
   แล้วกด "เข้าใจแล้ว" เพื่อปิดหน้าต่างและกลับมาที่หน้าตั้งค่า) หรือกด **"เริ่มเล่นเลย!"** เพื่อเข้าเกมทันที
3. **หน้าเล่นเกม** — นับถอยหลัง 3-2-1 แล้วประลองกันจนกว่าเวลาจะหมด (ตอบได้ต่อเนื่องหลายข้อ)
   เปิด/ปิดกล้องระหว่างเล่นได้จากปุ่ม 📷 บนแถบควบคุม
4. **หน้าสรุปผล** — ประกาศทีมชนะ คะแนน สถิติ พร้อมปุ่มเล่นอีกครั้ง

## 3. กติกา (Game Rules)

| เหตุการณ์ | คะแนน |
|---|---|
| ตอบถูกเป็นทีมแรก | +10 |
| แตะกล่องปริศนา (ต่อครั้ง ในเวลาสั้น ๆ หลังตอบถูก) | +1 (สุ่ม +2 เมื่อได้ "โบนัส x2") |
| แตะดาวทอง ⭐ ก่อนอีกทีม | +3 |
| ตอบผิด | หมดสิทธิ์เฉพาะข้อนั้น |

- ทั้งแมตช์จับเวลารวมตามที่ตั้งไว้ (60/90/120 วินาที) ตอบได้ต่อเนื่องหลายข้อจนกว่าเวลาจะหมด
- ถ้าไม่มีทีมตอบเลยเกิน ~20 วินาทีต่อข้อ (กันเกมค้าง) ระบบเฉลยแล้วขึ้นข้อใหม่ให้อัตโนมัติ
- การเลือกคำตอบ**ตอบติดทันที**ที่มือแตะเข้าลูกบอลคำตอบ (ไม่ต้องค้างมือ) ลูกบอลจะเรืองแสงให้เห็นว่ากำลังจะเลือกอันไหน — พื้นที่จับกว้างกว่าตัวลูกบอลจริงเพื่อให้กดง่ายขึ้น
- ตำแหน่งลูกบอลคำตอบสุ่มใหม่ทุกข้อ (ฝั่งละ 2 ลูก แบ่งครึ่งบน/ล่างไม่ให้ชนกัน) ป้องกันการจดจำตำแหน่งแล้วกดมั่ว
- ตอบถูกแล้วกล่องปริศนาจะโผล่ขึ้นมาให้ทีมนั้นแตะรัว ๆ ภายในเวลาสั้น ๆ (ยกมือไปจ่อที่กล่องในโหมดกล้อง หรือแตะ/กด Space ในโหมดแตะจอ)
- จบเกมแสดงทีมชนะ คะแนนรวม จำนวนข้อที่ตอบถูก และจำนวนครั้งที่เก็บกล่องปริศนาของแต่ละทีม

## 4. ระดับความยาก

| ระดับ | บวก/ลบ | คูณ | หาร | เหมาะกับ |
|---|---|---|---|---|
| 🌱 ง่าย | ผลลัพธ์ไม่เกิน 20 | แม่ 2–5 | ตัวหาร 2–5 | ป.1–2 |
| 🌿 ปานกลาง | ไม่เกิน 100 | แม่ 2–9 | ตัวหาร 2–9 | ป.2–4 |
| 🌳 ยาก | เลข 2–3 หลัก | แม่ 3–12 | ตัวหาร 3–12 | ป.4–6 |

ตัวลวง (distractor) สร้างให้ใกล้เคียงคำตอบจริง เช่น ±1, ±10 หรือผลคูณของแม่ข้างเคียง
เพื่อให้นักเรียนต้องคิดจริง ไม่สามารถเดาจากความห่างของตัวเลขได้

## 5. เทคโนโลยี

- **HTML ไฟล์เดียว** (`index.html`) — HTML + CSS + JavaScript ครบในไฟล์เดียว แจกจ่ายง่าย
- **MediaPipe PoseLandmarker** (tasks-vision 0.10.3 ผ่าน CDN) — ตรวจจับร่างกาย **2 คนพร้อมกัน**
  (`numPoses: 2`) แล้วแบ่งฝั่งซ้าย/ขวาจากตำแหน่งบนจอ
- ตำแหน่ง "มือ" คำนวณจากจุด landmark ข้อศอก→ข้อมือ (13→15, 14→16) ยืดปลายออกเล็กน้อย
  แบบเดียวกับเกมต้นแบบ — ใช้ตำแหน่งมือนี้ทั้งเลือกคำตอบและแตะกล่องปริศนา ผ่านตัวกรองความสั่น (1€ Filter)
  ให้ตำแหน่งนิ่งขึ้นโดยไม่หน่วง — **ไม่แสดงจุดเคอร์เซอร์บนจอ** ให้ลูกบอลคำตอบเรืองแสงเองเมื่อมือแตะเข้าไปแทน
  (ตัดปัญหาเห็นจุดสั่นไปในตัว) และตอบติดทันทีไม่ต้องค้างมือ (ไม่ใช้ dwell)
- เสียงเอฟเฟกต์และ**เพลงประกอบ** (ลูป 8 ห้อง สไตล์ chiptune) สังเคราะห์ด้วย **Web Audio API** — ไม่ต้องใช้ไฟล์เสียง มีปุ่ม 🎵 เปิด/ปิดเพลงแยกจากปุ่ม 🔊 เสียงรวม
- **พื้นหลังในเกม**: มีชั้น `#gameBg` ให้ใส่ภาพประกอบของตัวเอง (ดูหัวข้อ 8) ถ้ายังไม่มีไฟล์ภาพ ระบบจะข้ามไปใช้พื้นหลังไล่เฉดสีเดิมให้อัตโนมัติ ไม่มีภาพแตก
- **โหมดสำรองอัตโนมัติ**: ถ้ากล้องไม่พร้อม/ไม่อนุญาต เล่นด้วยการแตะจอหรือคีย์บอร์ดได้ทันที
  (ทีมแดง A/Z · ทีมน้ำเงิน K/M · แตะกล่องปริศนาด้วยเมาส์/นิ้ว หรือกด Space · เก็บดาว ⭐ ทีมแดงกด S ทีมน้ำเงินกด L)
- ตั้งค่าเก็บไว้ใน `localStorage` ของเบราว์เซอร์ (หมวดโจทย์ ระดับ ระยะเวลา เสียง เพลง) รอบเล่นถัดไปไม่ต้องตั้งใหม่
- รองรับ `prefers-reduced-motion` ปิดแอนิเมชันสำหรับเครื่องที่ตั้งค่าลดการเคลื่อนไหว

## 6. การใช้งานในห้องเรียน

1. ต่อคอมพิวเตอร์/โน้ตบุ๊กที่มีเว็บแคมเข้ากับทีวีหรือโปรเจกเตอร์
2. เปิดเกมผ่านเบราว์เซอร์ (Chrome/Edge แนะนำ) และอนุญาตให้ใช้กล้อง
3. ให้นักเรียน 2 ทีมยืนห่างจอประมาณ 1.5–2.5 เมตร คนละฝั่งของกล้อง
4. เพื่อนในทีมช่วยกันคิดเลขแล้วบอกตัวแทน — ทั้งห้องได้มีส่วนร่วม
5. สลับตัวแทนทุกรอบเกม เพื่อให้ทุกคนได้เล่น

> เคล็ดลับ: ที่ที่แสงสว่างสม่ำเสมอและฉากหลังไม่วุ่นวาย จะช่วยให้การตรวจจับแม่นยำขึ้น

## 7. การนำขึ้น GitHub Pages (เหมือนเกมต้นแบบ)

1. สร้าง repository ใหม่บน GitHub เช่น `KrukungGame-MathDuel`
2. อัปโหลด `index.html`, `cover.jpg`, โฟลเดอร์ `image/` (ถ้ามีภาพพื้นหลังในเกม) และ `README.md` เข้า branch `main`
3. ไปที่ Settings → Pages → Source เลือก `main` / root แล้วกด Save
4. รอสักครู่ เกมจะออนไลน์ที่ `https://<ชื่อผู้ใช้>.github.io/KrukungGame-MathDuel/`

หมายเหตุ: กล้องบนเว็บทำงานได้เฉพาะ `https://` หรือ `localhost` — GitHub Pages เป็น https อยู่แล้ว

## 8. การปรับแต่งเพิ่มเติม

แก้ไขในไฟล์ `index.html` ได้โดยตรง:

- **ใส่ภาพพื้นหลังในเกมของตัวเอง**: สร้างโฟลเดอร์ `image/` ไว้ข้าง ๆ `index.html` แล้ววางไฟล์ภาพแนวนอน
  (แนะนำ 1920×1080 หรืออัตราส่วนใกล้เคียง, สไตล์การ์ตูนสดใส ไม่ใช้ตัวละครมีลิขสิทธิ์)
  ตั้งชื่อไฟล์ตามอาร์เรย์ `BG_IMAGES` ในสคริปต์ (ค่าเริ่มต้น `image/bg1.jpg` ถึง `image/bg4.jpg`
  แก้รายชื่อ/จำนวนไฟล์ในอาร์เรย์นี้ได้เลย) ระบบจะสุ่มเลือก 1 ภาพต่อแมตช์ ถ้ายังไม่มีไฟล์ตามชื่อ
  จะข้ามไปใช้พื้นหลังไล่เฉดสีเดิมให้อัตโนมัติ
- ระยะเวลาแข่งที่เลือกได้ในหน้าตั้งค่า: ปุ่มในกลุ่ม `#segT` (ค่า `data-v` เป็นวินาที)
- เวลาสูงสุดที่รอต่อข้อก่อนเฉลยอัตโนมัติ (กันเกมค้าง): ค่า `QUESTION_SOFT_MS`
- ระยะพื้นที่จับ (นอกเหนือขนาดจริงของลูกบอล/ปุ่ม) เมื่อใช้กล้อง: ค่า `HOVER_PAD_RATIO` (สัดส่วนของขนาดปุ่ม)
- ระยะเวลาและอัตราการแตะกล่องปริศนา: ค่า `MBOX_WINDOW`, `MBOX_TAP_COOLDOWN`, `MBOX_DOUBLE_CHANCE`
- คะแนนแต่ละเหตุการณ์: ในฟังก์ชัน `tryAnswer`, `hitBox`, `grabStar`
- ช่วงตัวเลขโจทย์: ฟังก์ชัน `genQuestion`
